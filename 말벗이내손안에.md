## 2. 원본 프로젝트 분석 (malbeot-diary)

### 기술 스택
| 영역 | 원본 기술 | 신규 기술 |
|------|-----------|-----------|
| 백엔드 | FastAPI + SQLAlchemy | FastAPI + SQLAlchemy (선별 재작성) |
| 데이터베이스 | SQLite(개발) / PostgreSQL(운영) | SQLite(개발) / PostgreSQL(운영) |
| 프론트엔드 | 바닐라 HTML/CSS/JS | **React (Vite) + Tailwind CSS** |
| 인증 | JWT | JWT |
| AI | OpenAI GPT | **Claude API (Anthropic)** |
| 음성 | Azure STT/TTS | Azure STT/TTS (유지) |
| 알림 | APScheduler + WebPush | APScheduler + WebPush (유지) |
| 캐시 | Redis | Redis |
| 배포 | Docker + Nginx + EC2 | Docker + Nginx + EC2 |

### 원본 API 구조
```
/api/v1/
├── /auth      — 로그인, 회원가입, JWT 인증
├── /diaries   — 일기 CRUD
├── /personas  — AI 페르소나 설정
├── /feedback  — AI 공감 피드백
├── /voice     — 음성 입력(STT) / 음성 출력(TTS)
├── /alarms    — 알림 스케줄
└── /search    — AI 검색
```

### 원본 데이터 모델 (핵심)
```
Diary
├── id (UUID)
├── user_id (FK → User)
├── persona_id (FK → Persona, 선택)
├── title (제목)
├── emotion (감정)
├── weather (날씨)
├── content (내용)
├── input_type (text | voice | mixed)
├── audio_file_url (음성파일 경로)
└── diary_date (일기 날짜)
```

### 원본 화면 목록
| 파일 | 화면 |
|------|------|
| login.html | 로그인 |
| index.html | 메인(일기 목록) |
| diary_write.html | 일기 쓰기 |
| diary_read.html | 일기 읽기 |
| my-diary.html | 내 일기 모음 |
| profile.html | 프로필 |
| ai-persona.html | AI 페르소나 설정 |
