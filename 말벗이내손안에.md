## 6. 주요 기능 상세

### 6-1. AI 피드백 (Claude Haiku)

- 일기 저장 시 자동으로 Claude API 호출
- 페르소나 설정에 따라 말투 변경
- 마크다운 기호(`#`, `**` 등) 제거 처리 (프롬프트 + 정규식)
- 프롬프트: 공감 위주, 3~4문장, 판단/조언 금지, 순수 텍스트 지시

```python
# backend/app/services/feedback_service.py
model = "claude-haiku-4-5-20251001"
max_tokens = 300
```

### 6-2. TTS 읽어주기

- Web Speech Synthesis API (브라우저 내장, 무료)
- `useTTS.js` 훅으로 18번/19번 화면 공통 관리
- localStorage에 ON/OFF 설정 저장 (기본값: ON)
- 헤더에 🔊/🔇 아이콘 + 파란색 토글 스위치
- 18번(상세) 진입 시 기존 피드백 자동 읽기
- 19번(작성) 저장 후 새 피드백 자동 읽기
- 페이지 이탈(← 뒤로가기) 시 즉시 중단

> ⚠️ **주의**: TTS/STT는 HTTPS 환경에서만 동작 (브라우저 보안 정책)

### 6-3. 음성 입력 (STT)

- Web Speech Recognition API (브라우저 내장, 무료)
- 한국어(ko-KR), 연속 인식, 실시간 중간 결과 표시
- 크롬/갤럭시/아이폰 지원
- 🎤 말로 쓰기 버튼 (빨간 애니메이션 = 녹음 중)

> ⚠️ **주의**: HTTPS 환경에서만 마이크 권한 허용

### 6-4. 페르소나 (말벗 설정)

| 타입 | 이름 | 스타일 |
|------|------|------|
| `empathy` | 공감형 말벗 | 따뜻한 할머니 스타일 |
| `advice` | 조언형 말벗 | 솔직한 친구 스타일 |
| `custom` | 직접 설정 | 이름+성격 직접 입력 |

- 일기 작성 시 말벗 선택 → 선택한 스타일로 Claude 피드백 생성
- 선택 없으면 기본(공감형) 스타일 적용

### 6-5. 주요 기술 이슈 해결 기록

| 이슈 | 원인 | 해결 |
|------|------|------|
| 403 Not Authenticated | FastAPI 307 리다이렉트 시 Authorization 헤더 소실 | `redirect_slashes=False` + 엔드포인트 URL에 `/` 명시 |
| 브라우저 자동완성 오작동 | 회원가입 닉네임이 로그인 아이디에 자동완성 | `autoComplete="off"`, `autoComplete="new-password"` 추가 |
| AI 피드백에 마크다운 기호 표시 | Claude가 `#`, `**` 등 마크다운 사용 | 프롬프트에 순수 텍스트 지시 + 정규식 후처리 |

## 8. HTTPS 설정 가이드 (음성 기능 활성화)

> STT(음성입력)와 TTS(읽어주기)는 브라우저 보안 정책상 HTTPS에서만 동작
> 도메인이 필요하며, 무료 DuckDNS 도메인 + Let's Encrypt SSL 조합 권장

### Step 1. DuckDNS 무료 도메인 발급

1. https://www.duckdns.org 접속
2. 구글 또는 깃허브 계정으로 로그인
3. `domains` 입력란에 원하는 이름 입력 (예: `malbeot`)
4. `add domain` 클릭
5. `malbeot.duckdns.org` 도메인 생성 확인
6. `current ip` 란에 서버 IP(`13.209.191.143`) 입력 후 `update ip` 클릭

> 결과: `malbeot.duckdns.org` → `13.209.191.143` 연결 완료

### Step 2. EC2 보안그룹 포트 443 열기

AWS 콘솔 → EC2 → 보안그룹 → 인바운드 규칙 편집
- 유형: `HTTPS`
- 포트: `443`
- 소스: `0.0.0.0/0`

### Step 3. 서버에 Certbot 설치

```bash
sudo apt update
sudo apt install -y certbot
```

### Step 4. SSL 인증서 발급

```bash
# HumanRM nginx 잠시 중단 (80포트 사용 중이므로)
docker stop humanrm-nginx-1

# Let's Encrypt 인증서 발급 (도메인 확인에 80포트 사용)
sudo certbot certonly --standalone -d malbeot.duckdns.org

# HumanRM nginx 재시작
docker start humanrm-nginx-1
```

> 인증서 생성 위치:
> - `/etc/letsencrypt/live/malbeot.duckdns.org/fullchain.pem`
> - `/etc/letsencrypt/live/malbeot.duckdns.org/privkey.pem`

### Step 5. nginx.conf HTTPS 설정

`frontend/nginx.conf` 수정:

```nginx
server {
    listen 80;
    server_name malbeot.duckdns.org;
    return 301 https://$host$request_uri;
}

server {
    listen 443 ssl;
    server_name malbeot.duckdns.org;

    ssl_certificate /etc/letsencrypt/live/malbeot.duckdns.org/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/malbeot.duckdns.org/privkey.pem;

    root /usr/share/nginx/html;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location /api/ {
        proxy_pass http://app:8000/api/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_read_timeout 60s;
    }
}
```

### Step 6. docker-compose.yml 포트 및 인증서 마운트 추가

```yaml
services:
  frontend:
    build: ./frontend
    ports:
      - "443:443"
    volumes:
      - /etc/letsencrypt:/etc/letsencrypt:ro
    depends_on:
      - app
    restart: unless-stopped
```

### Step 7. 재빌드 및 재시작

```bash
cd ~/malbeot-diary2
docker compose down
docker compose up -d --build
```
