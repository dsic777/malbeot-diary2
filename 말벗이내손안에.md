## 6. 주요 기능 상세

### 6-1. AI 피드백 (Claude Haiku)

- 일기 저장 시 자동으로 Claude API 호출
- 페르소나 설정에 따라 말투 변경
- 마크다운 기호(`#`, `**` 등) 제거 처리 (프롬프트 + 정규식)
- 프롬프트: 공감 위주, 3~4문장, 판단/조언 금지, 순수 텍스트 지시

```python
# backend/app/services/feedback_service.py
model = "claude-haiku-4-5-20251001"
max_tokens = 300
```

### 6-2. TTS 읽어주기

- Web Speech Synthesis API (브라우저 내장, 무료)
- `useTTS.js` 훅으로 전 화면 공통 관리
- localStorage에 ON/OFF 설정 저장 (기본값: ON)
- 헤더에 🔊/🔇 아이콘 + 파란색 토글 스위치
- 기록 보기 진입 시 기존 피드백 자동 읽기
- 이야기 남기기 저장 후 새 피드백 자동 읽기
- 페이지 이탈(← 뒤로가기) 시 즉시 중단

> ⚠️ **주의**: TTS/STT는 HTTPS 환경에서만 동작 (브라우저 보안 정책)

### 6-3. 음성 입력 (STT)

- Web Speech Recognition API (브라우저 내장, 무료)
- 한국어(ko-KR), 실시간 중간 결과 표시
- 크롬/갤럭시/아이폰 지원
- 🎤 말로 쓰기 버튼 (빨간 애니메이션 = 녹음 중)
- **제목 음성 입력**: 제목 필드에도 🎤 버튼 추가 (제목/내용 독립 동작)
- **중지 버튼 누를 때까지 자동 재시작** (발화 단위로 처리)

> ⚠️ **주의**: HTTPS 환경에서만 마이크 권한 허용

#### Android 음성 중복 문제 해결 과정 (상세)

Android Chrome에서 음성 입력 시 텍스트가 2~5회 중복 입력되는 문제가 발생했다.

**원인 분석**

Web Speech Recognition API에서 `continuous: true` 설정 시 Android Chrome은 내부적으로 짧은 발화 단위마다 세션을 자동 종료/재시작한다. 재시작할 때 `event.resultIndex`가 0으로 리셋되면서 이전 발화 결과를 다시 전송하여 중복이 발생한다. 데스크톱 Chrome에서는 `continuous: true`가 정상 동작하므로 PC 테스트에서는 발견되지 않았다.

**1차 시도 — processedCountRef (실패)**

```javascript
// 처리된 resultIndex를 추적하여 skip
for (let i = event.resultIndex; i < event.results.length; i++) {
  if (i < processedCountRef.current) continue
  ...
}
```
→ Android 내부 재시작 시 `resultIndex`가 0으로 리셋되므로 효과 없음

**2차 시도 — sessionFinalRef 누적 텍스트 비교 (실패)**

```javascript
// 세션 전체 final 텍스트를 모아 이전 길이보다 늘어난 부분만 추가
if (allFinal.length > sessionFinalRef.current.length) {
  const newPart = allFinal.slice(sessionFinalRef.current.length)
  ...
}
```
→ Android 재시작 후 새 세션의 텍스트가 이전 누적 텍스트보다 짧아질 수 있어 slice 위치가 틀어짐. 여전히 중복 발생

**3차 시도 — continuous: false + onend 자동재시작 (성공)** ✅

```javascript
recognition.continuous = false   // 발화 1회 = 1세션, 중복 원천 차단
recognition.onresult = (event) => {
  // isFinal 결과만 깔끔하게 수신, 중복 없음
}
recognition.onend = () => {
  if (keepListeningRef.current) {
    recognition.start()  // 중지 버튼 누를 때까지 자동 재시작
  }
}
```

**핵심**: `continuous: false`로 발화 1회 = 1세션으로 분리하면 중복 자체가 불가능하다. `onend`에서 자동 재시작하므로 사용자 경험(계속 듣기)은 동일하게 유지된다. Android Chrome에서 Web Speech API를 안정적으로 사용하는 표준 패턴이다.

### 6-4. 페르소나 (말벗 설정)

| 타입 | 이름 | 스타일 |
|------|------|------|
| `empathy` | 공감형 말벗 | 따뜻한 할머니 스타일 |
| `advice` | 조언형 말벗 | 솔직한 친구 스타일 |
| `custom` | 직접 설정 | 이름+성격 직접 입력 |

- 일기 작성 시 말벗 선택 → 선택한 스타일로 Claude 피드백 생성
- 선택 없으면 기본(공감형) 스타일 적용

### 6-5. 캘린더

- 목록 화면 상단 탭으로 전환 (📋 목록 / 📅 캘린더)
- 날짜 셀에 감정 이모지 + 날씨 이모지 + 기록 건수 표시
- 감정색 20% 투명도로 날짜 셀 배경 힌트
- 날짜 클릭 → 해당 날짜 일기 목록 인라인 표시 → 상세로 이동
- 👈 👉 버튼 또는 좌우 스와이프/마우스 드래그로 월 이동

### 6-6. 검색

- 목록 화면 검색바에서 제목 + 내용 키워드 검색
- Enter 키 또는 검색 버튼 클릭
- ✕ 버튼으로 초기화, 검색 결과 건수 표시

### 6-7. 감정/날씨 UI

- 감정 버튼: 8종, 선택 시 감정별 고유 밝은 색 배경 + 검은 글씨
- 날씨 버튼: 5종, 선택 시 날씨별 고유 색상
- 기본값: 감정=평온(초록), 날씨=맑음(노랑)
- 목록 카드 우측에 입력방식(📝/🎤) + 감정 이모지 + 날씨 이모지 표시

### 6-8. 커스텀 날짜 피커

- 날짜 필드 클릭 시 커스텀 달력 팝업
- 선택된 날짜 옆에 요일 표시 (예: 2026-03-30 (월요일))
- 미래 날짜 선택 가능 → 스케줄/메모 용도로 활용
- 기본값: 로컬(한국) 오늘 날짜 (UTC 버그 수정)
- 좌우 스와이프/드래그로 월 이동

### 6-9. 일기 수정

- 기록 보기 헤더에 ✏️ 수정 버튼
- 클릭 시 이야기 남기기 화면으로 기존 내용 불러옴
- 타이틀: "이야기 수정하기", 저장 버튼: "✅ 수정 완료"
- 수정 완료 후 상세 화면으로 복귀 (피드백 재생성 없음)

### 6-10. 주요 기술 이슈 해결 기록

| 이슈 | 원인 | 해결 |
|------|------|------|
| 403 Not Authenticated | FastAPI 307 리다이렉트 시 Authorization 헤더 소실 | `redirect_slashes=False` + 엔드포인트 URL에 `/` 명시 |
| 브라우저 자동완성 오작동 | 회원가입 닉네임이 로그인 아이디에 자동완성 | `autoComplete="off"`, `autoComplete="new-password"` 추가 |
| AI 피드백에 마크다운 기호 표시 | Claude가 `#`, `**` 등 마크다운 사용 | 프롬프트에 순수 텍스트 지시 + 정규식 후처리 |
| 하단 버튼 사라짐 | `min-height: 100svh`라서 콘텐츠 많으면 페이지 무한 확장 | `height: 100svh` + `overflow: hidden` + main에 `overflow-y: auto` |
| 날짜 하루 전날 표시 | `toISOString()`이 UTC 기준 반환 (한국은 UTC+9) | `getFullYear/getMonth/getDate`로 로컬 날짜 직접 계산 |
| **Android 음성 중복 입력** | `continuous: true` 사용 시 Android Chrome이 내부 재시작하며 이전 결과 재전송 | `continuous: false` + `onend` 자동재시작 방식으로 전환 (상세 내용: 6-3 참조) |
| 날씨 버튼 선택 색상 안 보임 | 흐림/비/눈 색상이 어두운 배경에서 식별 어려움 | 더 선명한 색상으로 변경 (흐림: slate-400, 비: blue-400, 눈: sky-300, 바람: cyan-400) |
| 캘린더 복수 일기 아이콘 혼선 | 같은 날 여러 일기 시 첫 번째 일기의 감정/날씨 표시 | 마지막 입력 일기의 감정/날씨 표시로 변경 (`entries[entries.length-1]`) |
| DuckDNS 도메인 선점 | `malbeot.duckdns.org` 이미 타인이 사용 중 | `mymalbeot.duckdns.org` 로 대체 등록 |

## 8. HTTPS 설정 가이드 (음성 기능 활성화) ✅ 완료

> STT(음성입력)와 TTS(읽어주기)는 브라우저 보안 정책상 HTTPS에서만 동작
> 도메인이 필요하며, 무료 DuckDNS 도메인 + Let's Encrypt SSL 조합 사용

**적용 결과**: `https://mymalbeot.duckdns.org` — SSL 인증서 2026-06-27 만료, 자동갱신 등록 완료

### Step 1. DuckDNS 무료 도메인 발급

1. https://www.duckdns.org 접속
2. 구글 또는 깃허브 계정으로 로그인
3. `domains` 입력란에 원하는 이름 입력 (예: `mymalbeot`)
4. `add domain` 클릭
5. `current ip` 란에 서버 IP(`13.209.191.143`) 입력 후 `update ip` 클릭

> ⚠️ 인기 있는 이름은 이미 선점되어 있을 수 있음. `malbeot`은 선점됨 → `mymalbeot`으로 대체

### Step 2. EC2 보안그룹 포트 443 열기

AWS 콘솔 → EC2 → 보안그룹 → 인바운드 규칙 편집
- 유형: `HTTPS`, 포트: `443`, 소스: `0.0.0.0/0`

### Step 3. 서버에 Certbot 설치

```bash
sudo apt update && sudo apt install -y certbot
```

### Step 4. SSL 인증서 발급

```bash
# HumanRM nginx 잠시 중단 (80포트 사용 중이므로)
docker stop humanrm-nginx-1

# Let's Encrypt 인증서 발급 (도메인 확인에 80포트 사용)
sudo certbot certonly --standalone -d mymalbeot.duckdns.org

# HumanRM nginx 재시작
docker start humanrm-nginx-1
```

> 인증서 생성 위치:
> - `/etc/letsencrypt/live/mymalbeot.duckdns.org/fullchain.pem`
> - `/etc/letsencrypt/live/mymalbeot.duckdns.org/privkey.pem`

### Step 5. nginx.conf HTTPS 설정

`frontend/nginx.conf` 수정:

```nginx
server {
    listen 80;
    server_name mymalbeot.duckdns.org;
    return 301 https://$host$request_uri;
}

server {
    listen 443 ssl;
    server_name mymalbeot.duckdns.org;

    ssl_certificate     /etc/letsencrypt/live/mymalbeot.duckdns.org/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/mymalbeot.duckdns.org/privkey.pem;

    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;

    root /usr/share/nginx/html;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location /api/ {
        proxy_pass http://app:8000/api/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_read_timeout 60s;
    }
}
```

### Step 6. docker-compose.yml 포트 및 인증서 마운트 추가

```yaml
services:
  frontend:
    build: ./frontend
    ports:
      - "80:80"
      - "443:443"
    volumes:
      - /etc/letsencrypt:/etc/letsencrypt:ro
    depends_on:
      - app
    restart: unless-stopped
```
